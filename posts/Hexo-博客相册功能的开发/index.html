galleries 设计

index.md

​ 给使用 layout 为 galleries 的 index.md 设计变量，这些变量在galleries.ejs 中会以 page.XXX 的形式读取：

  • title 标题
  • date 时间，但在这里我并没用到
  • type 页面属性
  • layout 布局属性
  • layout_style 布局风格，我这里设计了两种风格 block 和 card
  • galleries 相册列表，列表里的内容必须符合子文件夹的属性（我没有想到如何自动遍历子文件夹并获取相关参数的办法，只能这手动档了。）
    • title 相册标题
    • description 相册描述
    • cover 相册封面
    • cover_style 我这里没设置，可以控制封面图片的 style
  • comments 是否打开评论功能

gallery 设计

index.md

​ 同理，给使用 layout 为 gallery 的 index.md 设计变量，这些变量在gallery.ejs 中会以 page.XXX 的形式读取：

  • title 标题

  • date 时间，但在这里我并没用到

  • type 页面属性

  • layout 布局属性

  • description 相册描述

  • imgs 定义的一个图片类（暂且先设计一个变量）

gallery.ejs

​ 在 gallery.ejs 中调用这些变量：

  • 因为这个博客使用了 fancybox 便于轮播图片，所以将 <img> 放在 <a class="fancybox"> 下。
  • 相册的布局是瀑布流布局，自己写可太费劲了，引用了 Masonry - imagesLoaded progress (codepen.io) 中的瀑布流插件。

    • imagesloaded.pkgd.js 取自 https://unpkg.com/imagesloaded@4/imagesloaded.pkgd.js
  • 设计了一个 description-container 用于存放图片的描述。
